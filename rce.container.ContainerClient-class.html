Class ContainerClient

source code

Container client is responsible for the creation and destruction of containers in a machine.

There can be only one Container Client per machine.

(Constructor)

source code 

Initialize the Container Client.

Parameters:
  • reactor (twisted::reactor) - Reference to the twisted reactor.
  • masterIP (str) - IP address of the Master process.
  • masterPort (int) - Port of the Master process used for internal communications.
  • masterPasswd (str) - SHA 256 Digested Master Password.
  • infraPasswd (str) - SHA 256 Digested Infra Password.
  • intIP (str) - IP address of the network interface used for the internal communication.
  • bridgeIP (str) - IP address of the network interface used for the container communication.
  • envPort (int) - Port where the environment process running inside the container is listening for connections to other endpoints. (Used for port forwarding.)
  • rosproxyPort (int) - Port where the rosproxy process running inside the container is listening for connections to console clients. (Used for port forwarding.)
  • rootfsDir (str) - Filesystem path to the root directory of the container filesystem.
  • confDir (str) - Filesystem path to the directory where container configuration files should be stored.
  • dataDir (str) - Filesystem path to the directory where temporary data of a container should be stored.
  • pkgDir ([(str, str)]) - Filesystem paths to the package directories as a list of tuples where each tuple contains the path to the directory in the host machine and the path to the directory to which the host directory will be bound in the container filesystem (without the @param rootfsDir).
