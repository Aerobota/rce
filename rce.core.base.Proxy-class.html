Class Proxy

source code

The Proxy should be used to represent an object from a slave process in the Master. It provides the same methods as the twisted.spread.pb.RemoteReference. Additionally, the Proxy is callable to get a Deferred which fires as soon as the RemoteReference or a Failure is present.

__destroy(self)

source code 

Method to destroy Proxy. Takes care of whether it's been called from the remote side or the local side. Calls __notify to clear up pending callbacks of callRemote or notifyOnDeath. Calls remote reference's (self.__obj) destroy function to make sure object on the remote side is also dead

callRemote(self, _name, *args, **kw)

source code 

Make a call to the RemoteReference and return the result as a Deferred. It exists to allow queuing of calls to remote reference before the remote reference has arrived.

For more information refer to twisted.spread.pb.RemoteReference.

Parameters:
  • _name (str) - Name of the method which should be called. The prefix 'remote_' will be added to the name in the remote object to select the method which should be called.
  • *args - Positional arguments which will be passed to the remote method.
  • **kw - Keyworded arguments which will be passed to the remote method.
Returns: twisted.internet.defer.Deferred
Deferred which will fire with the result of the call or a Failure if there was a problem.

callback(self, obj)

source code 

Register the remote reference which provides the necessary methods for this Proxy. Fires all pending callbacks passing on the remote reference as a parameter.

Exactly one call can be made to either 'callback' or 'errback'!

Parameters:
  • obj (twisted.spread.pb.RemoteReference) - Remote reference which should be registered.
