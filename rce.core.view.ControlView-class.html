Add a node to a ROS environment.

Parameters:

  • cTag (str) - Tag which is used to identify the ROS environment to which the node should be added.
  • nTag (str) - Tag which is used to identify the node in subsequent requests.
  • pkg (str) - Name of ROS package where the node can be found.
  • exe (str) - Name of executable (node) which should be launched.
  • args (str) - Additional arguments which should be used for the launch. Can contain the directives $(find PKG) or $(env VAR). Other special characters as '$' or ';' are not allowed.
  • name (str) - Name of the node under which the node should be launched.
  • namespace (str) - Namespace in which the node should be started in the environment.

view_addInterface(self, user, eTag, iTag, iType, clsName, addr='')

source code 

Add an interface to an endpoint, i.e. a ROS environment or a Robot object.

Parameters:
  • user (rce.core.user.User) - User for which the interface will be added.
  • eTag (str) - Tag which is used to identify the endpoint to which the interface should be added; either a container tag or robot ID.
  • iTag (str) - Tag which is used to identify the interface in subsequent requests.
  • iType (str) - Type of the interface. The type consists of a prefix and a suffix.
    • Valid prefixes are: ServiceClient, ServiceProvider, Publisher, Subscriber
    • Valid suffixes are: Interface, Converter, Forwarder
  • clsName (str) - Message type/Service type consisting of the package and the name of the message/service, i.e. 'std_msgs/Int32'.
  • addr (str) - ROS name/address which the interface should use. Only necessary if the suffix of @param iType is 'Interface'.
